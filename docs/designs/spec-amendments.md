## Accepted Scope Expansions (add to Wave 1)

### 1. `paygate demo` command
Add to §9 CLI:
```
paygate demo             Spin up demo echo server + testnet wallet, run full payment cycle
```
Specification: Starts a built-in echo server, requests testnet tokens from faucet, performs a TIP-20 transferWithMemo, verifies payment through the gateway, prints results with timing. Requires `PAYGATE_TEST_KEY` env var or prompts interactively.

### 2. Receipt verification endpoint
Add to §12 or new §12.4:
```
GET /paygate/receipts/{tx_hash}
```
Returns: `{ tx_hash, payer_address, amount, endpoint, verified_at, status }` or 404.
Public endpoint on main port. Rate-limited (100 req/min per IP).

### 3. Pricing page generator
Add to §9 CLI:
```
paygate pricing --html   Generate static HTML pricing page
```
Outputs an HTML file with: endpoint table, prices, accepted token, example curl commands.

### 4. Webhook on payment
Add to §5.1 config:
```toml
[webhooks]
payment_verified_url = ""     # POST on every verified payment
timeout_seconds = 5           # fire-and-forget timeout
```
Payload: `{ event: "payment.verified", tx_hash, payer_address, amount, endpoint, timestamp }`.
Fire-and-forget: webhook failures are logged but never block the response pipeline.
URL validation: reject private IPs, localhost, and link-local addresses at config load.

### 5. Response cost header
Add to §3.2 middleware stack, Receipt Injector layer:
```
X-Payment-Cost: 0.001000     # decimal string, amount charged for this request
```

### 6. `paygate wallet` command
Add to §9 CLI:
```
paygate wallet           Show provider on-chain balance + 24h income summary
```
Queries Tempo RPC for token balance, queries SQLite for recent revenue.

## Architecture Amendments

### Null receipt handling (§4.2 step 3)
Add after "Fetch tx receipt from Tempo RPC":
- If receipt is null (tx not yet indexed): return **400** with `Retry-After: 1` header and body `{"error": "Transaction not yet indexed, retry shortly"}`.
- If RPC returns an error (timeout, 5xx, network): return **503** with `Retry-After: 2` header.

### RPC connection pool (new, add to §5.1)
```toml
[tempo]
rpc_pool_max_idle = 10       # HTTP connection pool size for RPC
rpc_timeout_ms = 5000        # per-request timeout for RPC calls
```

### SQLite writer backpressure (amend §8)
The mpsc channel for the SQLite writer task should be **bounded** (default capacity: 10,000). When the channel is full, new write requests return 503 to the caller. This prevents unbounded memory growth under extreme load.

### Config reload (add to §5)
Config is held in `Arc<ArcSwap<Config>>` (or equivalent atomic swap). On SIGHUP or file change, the new config is validated before swapping. Invalid new config is rejected (log error, keep old config). Config changes are logged with a diff summary.

### Config validation at startup (add to §5)
All config fields are validated at startup:
- Addresses: valid 40-char hex with 0x prefix
- Prices: non-negative decimal strings
- URLs: valid format, webhook URLs must not be private/localhost
- Required fields: upstream, rpc_url, provider address
Clear error messages naming the specific field and issue.

## Security Amendments

### HMAC constant-time comparison (§4.3 step 6)
Session HMAC verification MUST use constant-time comparison to prevent timing attacks.

### Webhook URL validation (new §10)
Add to security table:
| Threat | Mitigation |
|--------|-----------|
| **Webhook SSRF** | webhook_url validated at config load: reject private IPs (10.x, 172.16-31.x, 192.168.x), localhost, link-local (169.254.x), and non-HTTPS schemes |

## Data Model Amendments

### Add session payer index (§8)
```sql
CREATE INDEX idx_sessions_payer ON sessions(payer_address);
```
Required for "max 5 concurrent sessions per payer" enforcement.

### Request log retention
Add TTL-based cleanup for request_log table:
```sql
-- Run periodically (e.g., every hour)
DELETE FROM request_log WHERE created_at < strftime('%s', 'now', '-30 days');
```
Default retention: 30 days. Configurable via:
```toml
[storage]
request_log_retention_days = 30
```
