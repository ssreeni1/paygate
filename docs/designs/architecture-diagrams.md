## 2. Direct Payment Data Flow (with shadow paths)

```
REQUEST ──▶ RATE LIMIT ──▶ MPP CHECK ──▶ VERIFY TX ──▶ BIND PAYER ──▶ PROXY ──▶ RESPONSE
  │            │              │             │              │             │           │
  ▼            ▼              ▼             ▼              ▼             ▼           ▼
[no body?]  [over limit?] [has headers?] [null rcpt?]  [mismatch?]  [timeout?]  [5xx?]
[too big?]  [banned IP?]  [quote exp?]  [wrong amt?]  [replay?]    [refused?]  [empty?]
[10MB max]  [429]         [free ep?]    [wrong memo?] [403/409]    [502/504]   [pass-thru]
                          [402+quote]   [ambiguous?]
                                        [too old?]
                                        [400/503]

Shadow paths:
  NIL INPUT:    Empty body → requestHash uses "" → valid for GET/DELETE
  EMPTY INPUT:  Zero-length POST body → requestHash = keccak256("POST /path\n") → valid
  ERROR PATH:   RPC error → 503 + Retry-After (never silent failure)
```

## 3. Quote Lifecycle State Machine

```
                    ┌──────────┐
                    │ CREATED  │
                    │ (in DB)  │
                    └────┬─────┘
                         │
              ┌──────────┼──────────┐
              ▼                     ▼
       ┌────────────┐       ┌────────────┐
       │  CONSUMED  │       │  EXPIRED   │
       │ (payment   │       │ (TTL pass, │
       │  ref'd it) │       │  cleanup)  │
       └────────────┘       └────────────┘

  Transitions:
    CREATED → CONSUMED:  Payment references this quote_id (one-time)
    CREATED → EXPIRED:   TTL (default 300s) passes without consumption
    CONSUMED → (final):  No further transitions
    EXPIRED → (deleted): TTL cleanup task removes from DB

  Invalid transitions:
    CONSUMED → CONSUMED: Prevented by single-use enforcement
    EXPIRED → CONSUMED:  Payment with expired quote re-verifies at current price
```

## 5. Deployment Sequence

```
  1. Build release binary
     └─▶ cargo build --release

  2. Deploy with config
     └─▶ Copy binary + paygate.toml to server

  3. Verify testnet (pre-flight)
     └─▶ paygate test
         ├─▶ PASS → continue
         └─▶ FAIL → abort, check RPC/config

  4. Switch to mainnet
     └─▶ Update paygate.toml: rpc_url, chain_id, accepted_token

  5. Start gateway
     └─▶ paygate serve
         └─▶ Check: /paygate/health → all green

  6. Verify first payment
     └─▶ Send test payment, confirm in /paygate/receipts/{tx}

  7. Monitor
     └─▶ Prometheus: paygate_payments_verified_total increasing
     └─▶ Logs: no 500s, verification_ms < 100ms p99
```

```
  ┌───────────────────┐
  │ Issue detected     │
  └────────┬──────────┘
           │
  ┌────────▼──────────┐
  │ Severity?         │
  └────────┬──────────┘
           │
     ┌─────┼──────┐
     ▼            ▼
  ┌──────┐   ┌────────────┐
  │Minor │   │  Critical  │
  │(perf,│   │  (wrong    │
  │ logs)│   │   verify,  │
  └──┬───┘   │   data     │
     │       │   loss)    │
     ▼       └─────┬──────┘
  ┌──────────┐     │
  │ Hot-fix  │     ▼
  │ + deploy │  ┌──────────────┐
  └──────────┘  │ 1. Stop      │
                │    paygate    │
                │ 2. Deploy old │
                │    binary     │
                │ 3. Restart    │
                │ 4. Verify     │
                │    /health    │
                └──────────────┘

  Data safety:
    - SQLite file is intact through restarts
    - No migration rollback needed (schema is additive)
    - Upstream API works independently of PayGate
    - Worst case: stop PayGate entirely, upstream serves directly
```
