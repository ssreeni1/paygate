## Chaos Scenarios

| Scenario | Expected Behavior |
|----------|-------------------|
| Kill -9 during payment verification | Consumer retries, tx_hash not consumed, re-verification succeeds |
| Tempo RPC down for 10 minutes | All direct payments → 503. Health endpoint reports "tempo_rpc: disconnected". Sessions (if Wave 2) continue working. |
| SQLite file deleted while running | Next write fails → 503. Health endpoint reports "db: error". Restart recreates schema. Payments since last backup lost. |
| 10K concurrent payment requests | Connection pool handles up to pool_max. Beyond that → 503 with Retry-After. SQLite writer bounded channel prevents memory blowup. |
| Webhook endpoint is slow (30s) | Fire-and-forget with timeout (5s default). Does not block response pipeline. Logs timeout. |
| Config file edited while serving | Next config reload picks up changes atomically (Arc swap). In-flight requests use old config. Log diff. |
