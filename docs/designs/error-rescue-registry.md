## Critical Rules

1. **No bare `catch-all` error handlers.** Every error type is named and has a specific rescue action.
2. **Every rescued error logs full context:** tx_hash, payer_address, endpoint, verification_step, latency_ms.
3. **Every 503 includes `Retry-After` header** (1-5 seconds depending on cause).
4. **HMAC comparison must use constant-time comparison** to prevent timing attacks.
5. **Webhook failures never block the response pipeline.** Fire-and-forget with logging.
