# PayGate — Security & Threat Model

Generated by `/plan-ceo-review` on 2026-03-18

## Threat Matrix

| # | Threat | Likelihood | Impact | Mitigated | Mitigation |
|---|--------|------------|--------|-----------|------------|
| 1 | Replay attack (reuse tx_hash) | High | High | YES | UNIQUE constraint on tx_hash in SQLite |
| 2 | Front-running (steal someone's payment) | Medium | High | YES | X-Payment-Payer must match on-chain `from` |
| 3 | Stale tx reuse | Medium | Medium | YES | tx_expiry_seconds (default 300s) |
| 4 | Wrong payment amount | Low | High | YES | Verify on-chain amount >= endpoint price |
| 5 | Wrong recipient | Low | High | YES | Decode Transfer event `to` (not tx `to`) |
| 6 | Memo forgery / endpoint mismatch | Low | High | YES | keccak256("paygate" \|\| quoteId \|\| requestHash) |
| 7 | Private key exposure | Low | Critical | YES | Env var only, never in config or logs |
| 8 | Ambiguous transactions | Low | Medium | YES | Reject txs with multiple matching Transfer events |
| 9 | Finality / reorg risk | Very Low | Critical | YES | Tempo single-slot finality, no reorgs |
| 10 | Price change after payment | Medium | Medium | YES | Quote TTL (5 min), honored within window |
| 11 | Quote replay | Low | Medium | YES | Quote IDs single-use |
| 12 | DDoS via payment flood | Medium | Medium | YES | Per-payer rate limit (10 req/s) |
| 13 | 402 bombardment | High | Low | YES | IP-based rate limit (1000/min) |
| 14 | Session abuse | Medium | Medium | YES | Max 5 concurrent sessions per payer |
| 15 | Large payload attack | Medium | Medium | YES | 10MB request body limit |
| 16 | Economic DDoS | Low | Low | YES | Attacker pays real money — self-limiting |
| 17 | HMAC timing attack | Low | Medium | YES* | Constant-time comparison (added in review) |
| 18 | Webhook SSRF | Medium | High | YES* | URL validation: reject private/localhost IPs (added in review) |
| 19 | Receipt endpoint info leak | Low | Low | YES* | Public (tx data is on-chain anyway), rate-limited (added in review) |
| 20 | Header injection to upstream | Medium | Medium | YES | All X-Payment-* headers stripped |
| 21 | Admin endpoint exposure | Medium | High | YES | admin_listen bound to 127.0.0.1 only |

`*` = Identified and resolved during this review.

## Input Validation Requirements

For every new user input path:

| Input | Validation | Rejection |
|-------|-----------|-----------|
| X-Payment-Tx | Valid 66-char hex (0x + 64) | 400 "Invalid transaction hash" |
| X-Payment-Payer | Valid 42-char hex address (0x + 40) | 400 "Invalid payer address" |
| X-Payment-Quote-Id | Matches `qt_[a-zA-Z0-9]+` | 400 "Invalid quote ID" |
| Request body | <= max_request_body_bytes | 413 "Payload too large" |
| paygate.toml addresses | Valid checksummed hex | Startup error with field name |
| paygate.toml prices | Non-negative decimal string | Startup error with field name |
| webhook_url | Valid HTTPS URL, not private/localhost/link-local IP | Startup error |

## Secrets Management

| Secret | Storage | Rotation | Logged? |
|--------|---------|----------|---------|
| PAYGATE_PRIVATE_KEY | Env var | Manual (restart required) | NEVER |
| Session secrets | SHA-256 hash in SQLite | Per-session (auto) | NEVER (hash only) |
| HMAC session keys | Derived from session secret | Per-session | NEVER |

|----------|-------------|-------------|

| /paygate/health | Anyone | None (public) |
| /paygate/metrics | Localhost only | admin_listen binding |
| /paygate/receipts/{tx} | Anyone | None (public, rate-limited) |
