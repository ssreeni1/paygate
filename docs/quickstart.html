Quickstart

How to pay for a 402-gated API.

What 402 Payment Required means

You received an HTTP 402 response because the API you called requires a per-request micropayment. The API is gated by PayGate, a reverse proxy that verifies on-chain payments on the Tempo blockchain before forwarding your request.

The 402 response includes everything you need to pay: the price, the token address, the recipient wallet, and a quote_id that locks in the quoted price for 5 minutes.

Here is what a typical 402 response looks like:

HTTP/1.1 402 Payment Required

X-Payment-Required: true
X-Payment-Amount: 1000
X-Payment-Decimals: 6
X-Payment-Token: 0x...USDC
X-Payment-Recipient: 0x...Provider
X-Payment-Network: tempo-mainnet
X-Payment-Chain-Id: 4217
X-Payment-Quote-Id: qt_a1b2c3d4
X-Payment-Quote-Expires: 1710799200
X-Payment-Methods: direct,session

{
  "error": "payment_required",
  "message": "Send 0.001000 USDC to 0x...Provider on Tempo, then retry with X-Payment-Tx header.",
  "help_url": "https://ssreeni1.github.io/paygate/quickstart",
  "pricing": {
    "amount": "0.001000",
    "amount_base_units": 1000,
    "decimals": 6,
    "token": "0x...USDC",
    "recipient": "0x...Provider",
    "quote_id": "qt_a1b2c3d4",
    "quote_expires_at": "2026-03-18T12:00:00Z",
    "methods": ["direct", "session"]
  }
}

The JSON body is the authoritative pricing source. The X-Payment-* headers provide a machine-parseable summary. Amounts in headers are always in base units (integers); the body includes both decimal strings and base unit integers.

Option 1: Use the SDK (recommended)

The @paygate/sdk TypeScript SDK handles the entire 402 negotiation automatically: it detects a 402 response, sends the on-chain payment, and retries the request with proof of payment.

Install

npm install @paygate/sdk

Configure a Tempo wallet and create the client

import { PayGateClient } from '@paygate/sdk';
import { createClient, http, publicActions, walletActions } from 'viem';
import { Account, tempoActions } from 'viem/tempo';
import { tempoTestnet } from 'viem/chains';

// Create a Tempo wallet using viem/tempo
const account = Account.fromSecp256k1(process.env.TEMPO_PRIVATE_KEY!);

const tempoClient = createClient({
  account,
  chain: tempoTestnet,
  transport: http(),
})
  .extend(publicActions)
  .extend(walletActions)
  .extend(tempoActions());

// Wrap in PayGateClient for auto-pay behavior
const client = new PayGateClient({
  tempoClient,
  autoSession: true,        // auto-create sessions for high-frequency use
  sessionDeposit: '0.10',   // $0.10 USDC per session
});

Make requests

Use client.fetch() exactly like the standard fetch() API. Payment happens automatically when a 402 is received.

const response = await client.fetch(
  'https://api.example.com/v1/chat/completions',
  {
    method: 'POST',
    body: JSON.stringify({ model: 'gpt-4', messages: [...] }),
  }
);

const data = await response.json();
console.log(data);

Discover pricing without paying

const pricing = await client.getPricing('https://api.example.com');
// => { "POST /v1/chat/completions": { price: "0.005", token: "USDC" } }

Use as an AI agent tool

const paidApiTool = {
  name: 'call_paid_api',
  description: 'Call a micropayment-gated API. Payment is handled automatically.',
  parameters: { url: 'string', method: 'string', body: 'object' },
  execute: async ({ url, method, body }) => {
    return await client.fetch(url, { method, body: JSON.stringify(body) });
  },
};

Option 2: Pay manually with curl

If you are not using the SDK, you can complete the payment flow manually. This is useful for testing, debugging, or integrating from languages without an SDK.

Step 1. Get the 402 response

Send a request to the API. You will receive a 402 with pricing details.

curl -i https://api.example.com/v1/chat/completions \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4","messages":[{"role":"user","content":"hello"}]}'

Note the quote_id, amount, token, and recipient from the JSON body.

Step 2. Compute the payment memo

The payment must include a memo that binds it to your specific request. The memo is computed as: 

requestHash = keccak256(METHOD + " " + PATH + "\n" + BODY)
memo        = keccak256("paygate" + quoteId + requestHash)

METHOD is the uppercase HTTP method. PATH includes the query string. BODY is the raw request body (empty string for GET/DELETE). All inputs are UTF-8 encoded and concatenated as raw bytes.

Step 3. Send the TIP-20 payment on Tempo

Send a TIP-20 transferWithMemo transaction on the Tempo blockchain:

  • to: the recipient address from the 402 response
  • amount: at least the amount_base_units from the 402 response
  • memo: the bytes32 memo computed in Step 2

You can use viem, ethers, or any Ethereum-compatible library to send the transaction. The token contract supports the standard transferWithMemo(address to, uint256 amount, bytes32 memo) function.

Step 4. Retry with payment proof

Once the transaction is confirmed, resend your original request with two additional headers: 

curl -i https://api.example.com/v1/chat/completions \
  -X POST \
  -H "Content-Type: application/json" \
  -H "X-Payment-Tx: 0xYOUR_TX_HASH" \
  -H "X-Payment-Payer: 0xYOUR_WALLET_ADDRESS" \
  -H "X-Payment-Quote-Id: qt_a1b2c3d4" \
  -d '{"model":"gpt-4","messages":[{"role":"user","content":"hello"}]}'

PayGate will verify the payment on-chain (typically under 100ms) and forward your request to the upstream API.

Understanding the response headers

After successful payment verification, PayGate forwards your request to the upstream API and returns the response with additional headers:

Header Description
X-Payment-Receipt A unique receipt identifier for this paid request. You can look up the payment details at GET /paygate/receipts/{tx_hash}.
X-Payment-Cost The amount charged for this request, in base units (e.g., 1000 for $0.001000 USDC with 6 decimals).

All X-Payment-* headers are stripped before the request is forwarded to the upstream API. The upstream service sees a normal HTTP request.

Testnet testing

Before using real funds, test the full payment flow on the Tempo Moderato testnet.

Get testnet tokens

Fund your wallet with testnet tokens using the Tempo faucet RPC method: 

curl -X POST https://rpc.moderato.tempo.xyz \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tempo_fundAddress",
    "params": ["YOUR_ADDRESS"],
    "id": 1
  }'

Replace YOUR_ADDRESS with your wallet address. This will send testnet tokens to your account.

Run the end-to-end test

If you are running a PayGate instance, the built-in test command exercises the full payment cycle on testnet: 

paygate test

This starts an echo server, sends a request without payment, funds a test wallet, pays on-chain, verifies the payment, and checks replay protection.
